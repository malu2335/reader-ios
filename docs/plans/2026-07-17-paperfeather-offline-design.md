# “纸羽轻阅”纯本地阅读器设计

## 目标与取舍

应用从“本地优先、仍保留在线遗留与 AI 接口”收敛为严格的纯本地阅读器，并改名为“纸羽轻阅”。名称中的“纸羽”表达纸页与轻盈，“阅”明确阅读属性；不出现“本地”字样，也避开 App Store 已占用的“轻阅”和“简阅”。保留 TXT、EPUB、PDF、MOBI、ZIP/CBZ、图片文件夹导入，书架、阅读进度、书签、系统词典、系统朗读、自定义字体、正文净化及文件备份恢复。移除 AI 配置和翻译、在线书城/发现/搜索、远程配置、更新检查、在线书籍详情与章节请求、网页入口。

## 架构

采用三层防线。第一层是产品入口：主界面只保留书架与设置，设置和阅读页删除 AI，书架查询只返回 `bookId < 0` 的本地书。第二层是传输层：原 `RDBaseApi` 改为立即返回“离线版本不支持网络请求”的本地失败结果，移除 YTKNetwork/AFNetworking 与网页进度依赖；应用启动时注册 URL Loading 拦截器，拒绝所有 HTTP/HTTPS 请求，防止遗留不可达代码、图片加载器或未来误接入产生外联。第三层是工程与数据：删除 ATS 本地网络例外，备份不再写入或恢复 AI 配置，AI 与网页实现不进入 Sources 构建阶段。

## 兼容与验证

不删除用户已有数据库记录，但书架、计数和备份只读取本地书，避免破坏性迁移。旧备份中的 `ai_config.json` 被忽略，其余书籍、进度、书签、字体和净化规则继续兼容。验证包括 Pod 依赖检查、Xcode Release 编译、现有本地 harness、源代码 HTTP/HTTPS 扫描、PBX Sources 检查，以及品牌字符串和 Info.plist/LaunchScreen 一致性检查。网络遗留源码可保留用于历史追踪，但不会编译进 App；运行时拦截器作为最终不可绕过的应用级保护。
