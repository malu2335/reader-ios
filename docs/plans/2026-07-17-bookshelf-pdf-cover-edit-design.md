# PDF 首页封面与书架信息编辑设计

## 目标

- PDF 导入后用第一页生成书架封面，渲染失败时保留文字封面兜底。
- 已导入的旧 PDF 在后台补齐第一页封面，不阻塞书架主线程。
- 所有书架书籍（本地书和在线书）均可手动修改书名、作者和封面。
- 用户封面与在线书的远端 `coverImg` 分离，避免网络模型刷新覆盖本地选择。

## 方案

PDF 仍沿用 `RDBookDetailModel.coverImg` 保存自动封面文件名。导入队列使用 PDFKit 打开已经复制到 `Documents/LocalBooks` 的 PDF，读取第一页并渲染为固定书封画布；自动封面使用带版本的确定性文件名，便于识别旧文字封面并在后台迁移。损坏、加密或无页面的 PDF 不因封面生成失败而崩溃，继续使用现有文字封面。

手动封面不写入 `coverImg`，而是在 `LocalBooks` 中按 `bookId` 保存为确定性文件。这样正数在线书与负数本地书都可覆盖封面，也无需新增 WCDB 列。书架加载顺序为：手动封面、本地自动封面、在线封面、占位图。删除书籍或清空书架时同步删除手动封面；本地书备份同时包含手动封面。

每个 `bookId` 的选图请求都有全局代次。开始新选图、恢复默认、删除和备份恢复都会推进代次；代次校验、书架记录校验与文件写入在独立的手动封面串行队列中完成，防止慢速 iCloud 回调覆盖较新封面或在删除后重建孤儿文件。备份从同一队列取得封面数据快照，保证 JSON 清单与 ZIP 内容一致。

现有改名操作继续按列更新，避免用书架轻量模型整行覆盖阅读进度。在线书首次章节加载的整行写回要继承数据库中的书名和作者，使用户修改不会被网络模型覆盖。

## 交互与数据流

书架长按菜单保留“修改书名”，新增“更换封面”；已有手动封面时再显示“恢复默认封面”。封面选择使用 iOS 15 的 PHPicker，仅允许一张图片，不申请完整相册读取权限。选图完成后在后台归一化方向、居中裁切到书封比例、压缩并原子写入，主线程只负责提示和刷新。

旧 PDF 封面回填由书架后台加载流程调用，并串行复用本地书导入队列。Cell 和 `coverForBook:` 只读已经生成的图片，绝不现场渲染 PDF。重复导入旧 PDF 时也会触发封面修复。

## 错误处理

- PDF 首页不可渲染：继续使用文字封面，不影响导入和阅读。
- 相册取消：不修改任何数据。
- 图片读取或落盘失败：保留原封面并显示失败提示。
- 手动封面文件丢失：自动回退到本地自动封面或在线封面。
- 存量迁移失败：保留旧封面；后续后台加载可再次尝试。

## 验证

- 新导入 PDF 显示第一页；旧 PDF 启动后被回填。
- PDF、EPUB/TXT/漫画和在线书均可改名、换封面、再次换封面并恢复默认。
- 重启后书名和手动封面仍在；在线书刷新或首次加载章节后不被覆盖。
- HEIC、大图、横图和取消选择均正常。
- 单本删除、清空书架和本地备份恢复不会留下或丢失相关封面。
- 完成模拟器 Debug 构建，并复核工作区差异。
